# 🔧 Решение проблем

## Проблема: "Нет связи" в веб-интерфейсе

### Шаг 1: Проверьте Arduino

1. **Откройте Serial Monitor в Arduino IDE (115200 baud)**
   
   Должны видеть:
   ```json
   {"status":"STARTUP"}
   {"status":"CHECK","sensor":"ADXL345"}
   {"status":"OK","sensor":"ADXL345"}
   ...
   {"status":"READY"}
   {"t":12345,"d":1,"m":[...],"r":[...],...}
   ```

2. **Если не видите сообщений:**
   - Проверьте подключение USB
   - Проверьте скорость порта (115200)
   - Перезагрузите Arduino

3. **Если видите ошибки датчиков:**
   - Это нормально, если датчики не подключены
   - Главное - должно быть `{"status":"READY"}`

---

### Шаг 2: Проверьте сеть

1. **Ping Arduino:**
   ```bash
   ping 192.168.0.177
   ```
   
   Должен отвечать. Если нет:
   - Проверьте IP адрес в Arduino коде
   - Проверьте что Arduino и компьютер в одной сети
   - Проверьте Ethernet кабель

2. **Проверьте HTTP сервер:**
   ```bash
   curl http://192.168.0.177/
   ```
   
   Должен вернуть HTML страницу. Если нет:
   - Arduino не запустил HTTP сервер
   - Firewall блокирует порт 80
   - Ethernet модуль не работает

3. **Проверьте /data endpoint:**
   ```bash
   curl http://192.168.0.177/data
   ```
   
   Должен вернуть JSON:
   ```json
   {"rpm":[0,0,0,0],"temp":[25.5,26.1,...],"thrust":0.0,"vibration":{...},"noise":45.2}
   ```

---

### Шаг 3: Проверьте конфигурацию

1. **Откройте `config/config.json`**
   
   Проверьте:
   ```json
   {
     "connection": {
       "mode": "direct",
       "direct": {
         "arduino_ip": "192.168.0.177"  // ← Ваш IP Arduino
       }
     }
   }
   ```

2. **Если IP неверный:**
   - Измените на правильный
   - Сохраните файл
   - Перезагрузите страницу

---

### Шаг 4: Используйте тестовую страницу

1. **Откройте тестовую страницу:**
   ```
   http://localhost:8000/frontend/test.html
   ```

2. **Введите IP Arduino**

3. **Нажмите все кнопки по порядку:**
   - Проверить доступность
   - Получить данные
   - Открыть /
   - Проверить CORS
   - Показать информацию о сети

4. **Смотрите результаты:**
   - Зеленый = OK
   - Красный = Ошибка

---

### Шаг 5: Проверьте консоль браузера

1. **Откройте консоль (F12)**

2. **Перезагрузите страницу**

3. **Смотрите логи:**
   ```
   ✅ Конфигурация загружена: {...}
   🔗 URL подключения (direct): http://192.168.0.177:80/data
   📡 Запрос данных: http://192.168.0.177:80/data
   📥 Ответ получен: 200 OK
   ✅ Данные распарсены: {...}
   ```

4. **Если видите ошибки:**
   - `Failed to fetch` = Arduino недоступен
   - `CORS error` = Нет CORS заголовков
   - `JSON parse error` = Неверный формат ответа
   - `404 Not Found` = Endpoint не найден

## Частые проблемы

### 1. CORS ошибка

**Симптом:**
```
Access to fetch at 'http://192.168.0.177/data' from origin 'file://' has been blocked by CORS policy
```

**Решение:**
```bash
# Используйте локальный сервер:
python -m http.server 8000

# Откройте:
http://localhost:8000/frontend/index.html

# НЕ открывайте через file:///
```

---

### 2. Arduino не отвечает

**Симптом:**
```
Failed to fetch
net::ERR_CONNECTION_REFUSED
```

**Проверьте:**
1. Arduino включен и подключен к сети
2. IP адрес правильный
3. HTTP сервер запущен (смотрите Serial Monitor)
4. Firewall не блокирует

**Решение:**
```bash
# Проверьте доступность:
ping 192.168.0.177
curl http://192.168.0.177/

# Если не отвечает - проверьте Arduino код
```

---

### 3. Неверный формат данных

**Симптом:**
```
Unexpected token < in JSON at position 0
```

**Причина:**
Arduino отправляет HTML вместо JSON

**Решение:**
1. Проверьте что запрашиваете `/data`, а не `/`
2. Проверьте Arduino код - функция `sendHTTPData()`
3. Проверьте Serial Monitor - должен быть JSON

---

### 4. Данные не обновляются

**Симптом:**
Индикатор зеленый, но данные не меняются

**Проверьте:**
1. Serial Monitor - данные должны обновляться
2. Консоль браузера - запросы должны идти каждые 100ms
3. Датчики подключены и работают

**Решение:**
```javascript
// В консоли браузера:
console.log('Интервал обновления:', UPDATE_INTERVAL);
console.log('Последние данные:', chartData);
```

---

### 5. Конфигурация не загружается

**Симптом:**
```
⚠️ Не удалось загрузить config.json
```

**Причина:**
Неверный путь к файлу

**Решение:**
1. Убедитесь что структура:
   ```
   /
   ├── config/
   │   └── config.json
   └── frontend/
       ├── index.html
       └── app.js
   ```

2. Запускайте сервер из корня проекта:
   ```bash
   python -m http.server 8000
   ```

3. Открывайте:
   ```
   http://localhost:8000/frontend/index.html
   ```
