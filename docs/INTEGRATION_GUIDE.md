# 🔗 Руководство по интеграции Arduino и веб-интерфейса

## Для программиста Arduino (твой друг)

### Шаг 1: Определите свои переменные
Откройте файл `data-mapping.json` и найдите раздел `arduino_variables`. Там указаны названия переменных, которые нужны для веб-интерфейса.

**Пример:**
```json
"rpm": {
  "arduino_variables": [
    "motor1_rpm",    // <- Замените на свое название
    "motor2_rpm",
    "motor3_rpm",
    "motor4_rpm"
  ]
}
```

### Шаг 2: Заполните таблицу маппинга

Создайте таблицу соответствия между вашими переменными и требуемыми:

| Параметр | Ваша переменная | Тип данных |
|----------|----------------|------------|
| motor1_rpm | `ваше_название` | int |
| motor2_rpm | `ваше_название` | int |
| motor3_rpm | `ваше_название` | int |
| motor4_rpm | `ваше_название` | int |
| motor1_temp | `ваше_название` | float |
| motor2_temp | `ваше_название` | float |
| motor3_temp | `ваше_название` | float |
| motor4_temp | `ваше_название` | float |
| total_thrust | `ваше_название` | float |
| accel_x | `ваше_название` | float |
| accel_y | `ваше_название` | float |
| accel_z | `ваше_название` | float |
| noise_level | `ваше_название` | float |

### Шаг 3: Используйте шаблон
Откройте файл `arduino-template.ino` и:

1. Замените названия переменных в разделе "ПЕРЕМЕННЫЕ ДАТЧИКОВ" на свои
2. Скопируйте функцию `sendSensorData()` в свой код
3. Добавьте вызов `handleHTTPRequests()` в свой `loop()`

### Шаг 4: Настройте сеть
```cpp
IPAddress ip(192, 168, 1, 100);  // Измените на свой IP
```

### Шаг 5: Проверьте библиотеки
Убедитесь что установлены:
- `Ethernet.h` (встроенная)
- `ArduinoJson.h` (установить через Library Manager)

## Для программиста Frontend (ты)

### Шаг 1: Получите IP адрес Arduino
Спросите у друга какой IP адрес он установил для Arduino.

### Шаг 2: Обновите конфигурацию
Откройте `app.js` и измените:
```javascript
const ARDUINO_IP = '192.168.1.100'; // IP от друга
```

### Шаг 3: Проверьте структуру данных
Убедитесь что структура в `data-mapping.json` соответствует тому, что отправляет Arduino.

### Шаг 4: Тестирование
1. Arduino должен быть подключен к той же сети
2. Откройте `index.html` в браузере
3. Проверьте индикатор подключения (должен стать зеленым)

## Формат данных (для обоих)

Arduino должен отправлять JSON в таком формате:

```json
{
  "rpm": [1200, 1250, 1180, 1220],
  "temp": [45.5, 47.2, 44.8, 46.1],
  "thrust": 850.5,
  "vibration": {
    "x": 0.15,
    "y": 0.12,
    "z": 9.85
  },
  "noise": 75.3
}
```

### HTTP Endpoint
- **URL:** `http://<ARDUINO_IP>/data`
- **Метод:** GET
- **Content-Type:** application/json
- **ВАЖНО:** Заголовок `Access-Control-Allow-Origin: *` обязателен!

## Отладка

### Проблема: "Нет связи" в веб-интерфейсе

**Проверьте:**
1. Arduino подключен к сети и доступен по IP
2. Открыть в браузере: `http://<ARDUINO_IP>/data` - должен вернуть JSON
3. В консоли браузера (F12) проверьте ошибки CORS
4. Убедитесь что заголовок `Access-Control-Allow-Origin: *` присутствует

### Проблема: Данные не обновляются

**Проверьте:**
1. Arduino отправляет актуальные данные (проверьте Serial Monitor)
2. Интервал обновления в `app.js` (по умолчанию 100ms)
3. Формат JSON корректный (используйте JSON validator)

### Проблема: Некоторые данные не отображаются

**Проверьте:**
1. Названия полей в JSON совпадают с `data-mapping.json`
2. Типы данных корректные (массивы, числа, объекты)
3. Все поля присутствуют в ответе
