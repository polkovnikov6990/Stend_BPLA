## 🔧 arduino/

Код для Arduino Nano с поддержкой всех датчиков.

### `quadcopter_stand.ino`

**Назначение:** Основной скетч для Arduino

**Функциональность:**
- Чтение данных со всех датчиков
- HTTP сервер на порту 80
- Отправка данных в JSON формате
- Обработка прерываний для подсчета оборотов

**Используемые библиотеки:**
- `Ethernet.h` - работа с W5500
- `ArduinoJson.h` - формирование JSON
- `HX711.h` - работа с тензодатчиками
- `Wire.h` - I2C коммуникация
- `Adafruit_ADXL345_U.h` - акселерометр

**Endpoints:**
- `GET /` - домашняя страница
- `GET /data` - данные датчиков в JSON

**Конфигурация:**
Все настройки берутся из `config/config.json`:
- IP адрес и сетевые параметры
- Пины датчиков
- Калибровочные коэффициенты

**Формат ответа `/data`:**
```json
{
  "rpm": [1200, 1250, 1180, 1220],
  "temp": [45.5, 47.2, 44.8, 46.1],
  "thrust": 850.5,
  "vibration": {"x": 0.15, "y": 0.12, "z": 9.85},
  "noise": 75.3
}
```

## 🐍 backend/

Python сервер для расширенного мониторинга (опционально).

### `web_server.py`

**Назначение:** HTTP API сервер для веб-интерфейса

**Функциональность:**
- Получает данные от Arduino Monitor
- Преобразует в формат для веб-интерфейса
- Предоставляет REST API
- Обработка CORS

**Класс `WebAPIHandler`:**
- `do_GET()` - обработка GET запросов
- `do_OPTIONS()` - обработка CORS preflight
- `send_data_response()` - отправка данных датчиков
- `send_stats_response()` - отправка статистики
- `send_ping_response()` - проверка связи

**Класс `WebServer`:**
- `start()` - запуск сервера
- `stop()` - остановка сервера
- `_run_server()` - основной цикл

**Endpoints:**
- `GET /data` - данные датчиков
- `GET /stats` - статистика работы
- `GET /ping` - проверка связи

**Конфигурация:**
```python
server = WebServer(monitor, host='0.0.0.0', port=8080)
```

**Зависимости:**
- `http.server` - HTTP сервер
- `json` - работа с JSON
- `threading` - многопоточность

### `monitor_with_web.py`

**Назначение:** Интегрированная версия монитора с веб-сервером

**Функциональность:**
- Расширяет GUI оригинального монитора
- Добавляет управление веб-сервером
- Автоматическая интеграция компонентов

**Класс `MonitorGUIWithWeb`:**
- Наследует `MonitorGUI`
- Добавляет элементы управления веб-сервером
- `toggle_web()` - запуск/остановка сервера

**Использование:**
```bash
python monitor_with_web.py
```

**Требования:**
- Оригинальный скрипт Arduino Monitor
- `web_server.py`
- Все зависимости Python

## ⚙️ config/

Конфигурационные файлы проекта.

### `config.json`

**Назначение:** Единый конфигурационный файл для всего проекта

**Структура:**

#### 1. Информация о проекте
```json
{
  "project": {
    "name": "Стенд тестирования БПЛА",
    "version": "1.0.0",
    "authors": ["Полковников А.В.", "Глазырин К.С."]
  }
}
```

#### 2. Настройки подключения
```json
{
  "connection": {
    "mode": "direct",  // или "server"
    "direct": {
      "arduino_ip": "192.168.1.100",
      "arduino_port": 80
    },
    "server": {
      "server_ip": "localhost",
      "server_port": 8080
    }
  }
}
```

**Режимы:**
- `direct` - прямое подключение к Arduino
- `server` - через Python сервер

#### 3. Конфигурация Arduino

**Сетевые параметры:**
```json
{
  "arduino": {
    "network": {
      "mac": "DE:AD:BE:EF:FE:ED",
      "ip": "192.168.1.100",
      "gateway": "192.168.1.1",
      "subnet": "255.255.255.0"
    }
  }
}
```

**Пины датчиков:**
```json
{
  "pins": {
    "rpm_sensors": {
      "motor_1": 2,
      "motor_2": 3,
      "motor_3": 4,
      "motor_4": 5
    },
    "temperature_sensors": {
      "motor_1": "A0",
      "motor_2": "A1",
      "motor_3": "A2",
      "motor_4": "A3"
    },
    "load_cells": {
      "hx711_1_dout": 6,
      "hx711_1_sck": 7,
      "hx711_2_dout": 8,
      "hx711_2_sck": 9
    }
  }
}
```

**Калибровка:**
```json
{
  "calibration": {
    "temperature": {
      "r_series": 10000,
      "r_nominal": 100000,
      "b_coefficient": 3950
    },
    "load_cells": {
      "scale_1": 420.0,
      "scale_2": 420.0
    }
  }
}
```

#### 4. Настройки веб-интерфейса
```json
{
  "frontend": {
    "update_interval_ms": 100,
    "max_chart_points": 100,
    "theme": {
      "primary_color": "#92000a",
      "secondary_color": "#6b0f1a"
    }
  }
}
```

#### 5. Настройки бэкенда
```json
{
  "backend": {
    "uart": {
      "baudrate": 115200
    },
    "udp": {
      "port": 8888
    },
    "http": {
      "port": 8080
    }
  }
}
```

**Как использовать:**
1. Скопируйте `config.json` и измените параметры
2. Arduino автоматически использует эти настройки
3. Веб-интерфейс загружает конфиг при старте
4. Python сервер читает конфиг для настройки

## 🌐 frontend/

Веб-интерфейс для мониторинга.

### `index.html`

**Назначение:** Главная страница веб-интерфейса

**Структура:**
```html
<!DOCTYPE html>
<html>
  <head>
    <!-- Мета-теги, стили, Chart.js -->

  <body>
    <!-- Шапка кафедры -->
```

**Секции:**

## 🔄 Поток данных

### Прямое подключение (mode: "direct")

```
Arduino → HTTP → Browser
  ↓
quadcopter_stand.ino
  ↓
GET /data
  ↓
JSON Response
  ↓
app.js → updateUI()
  ↓
DOM Update
```

### Через сервер (mode: "server")

```
Arduino → UART/UDP → Python → HTTP → Browser
  ↓           ↓         ↓       ↓        ↓
Датчики → Monitor → web_server → app.js → DOM
```

## 📝 Формат данных

### От Arduino к веб-интерфейсу

```json
{
  "rpm": [1200, 1250, 1180, 1220],
  "temp": [45.5, 47.2, 44.8, 46.1],
  "thrust": 850.5,
  "vibration": {
    "x": 0.15,
    "y": 0.12,
    "z": 9.85
  },
  "noise": 75.3
}
```

**Типы данных:**
- `rpm` - массив из 4 целых чисел (RPM)
- `temp` - массив из 4 float (°C)
- `thrust` - float (граммы)
- `vibration` - объект с x, y, z (м/с²)
- `noise` - float (дБ)

## 🛠️ Расширение проекта

### Добавление нового датчика

1. **Arduino:**
   - Добавьте пин в `config/config.json`
   - Добавьте чтение в `readAllSensors()`
   - Добавьте в JSON в `sendSensorData()`

2. **Frontend:**
   - Добавьте карточку в `index.html`
   - Добавьте стили в `style.css`
   - Добавьте обработку в `app.js`

3. **Backend (если используется):**
   - Добавьте маппинг в `web_server.py`

### Изменение интервала обновления

В `config/config.json`:
```json
{
  "frontend": {
    "update_interval_ms": 200  // было 100
  }
}
```

### Изменение цветовой схемы

В `config/config.json`:
```json
{
  "frontend": {
    "theme": {
      "primary_color": "#YOUR_COLOR",
      "secondary_color": "#YOUR_COLOR"
    }
  }
}
```

Затем обновите `style.css` с новыми цветами.

## 🔐 Безопасность

- CORS включен для всех источников (`*`)
- Нет аутентификации (локальная сеть)
- Нет шифрования (HTTP, не HTTPS)

**Рекомендации:**
- Используйте только в изолированной сети
- Не подключайте к интернету напрямую
- Для продакшена добавьте аутентификацию

## 📈 Производительность

- **Arduino:** ~10 запросов/сек
- **Python сервер:** ~100 запросов/сек
- **Веб-интерфейс:** обновление каждые 100 мс
- **Память Arduino:** ~60% (зависит от библиотек)

---

## 🐛 Известные ограничения

1. **Arduino:**
   - Ограниченная память (2KB RAM)
   - Один клиент одновременно
   - Блокирующие операции

2. **Веб-интерфейс:**
   - История только 100 точек
   - Нет постоянного хранения
   - Требует активного подключения

3. **Python сервер:**
   - Требует постоянной работы
   - Дополнительная точка отказа
