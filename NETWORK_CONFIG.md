# 🌐 Сетевая конфигурация проекта

## Единая сеть: 192.168.0.x

Все устройства должны быть в одной подсети для корректной работы.

---

## 📋 Таблица IP адресов

| Устройство | IP адрес | Порт | Протокол | Описание |
|------------|----------|------|----------|----------|
| **Arduino Nano** | `192.168.0.177` | 80 | HTTP | Веб-сервер для прямого подключения |
| **Arduino Nano** | `192.168.0.177` | 8888 | UDP | Broadcast для Python сервера |
| **Python Backend** | `localhost` | 8080 | HTTP | API сервер (опционально) |
| **Frontend Server** | `localhost` | 8000 | HTTP | Локальный веб-сервер |

## 🔧 Настройка Arduino

### В коде Arduino (`arduino/quadcopter_stand.ino`):

```cpp
// MAC адрес (уникальный для каждого устройства)
const PROGMEM byte mac[] = {0xDE, 0xAD, 0xBE, 0xEF, 0xFE, 0x01};

// IP адрес Arduino
IPAddress ip(192, 168, 0, 177);

// Broadcast адрес для UDP
IPAddress broadcastIp(192, 168, 0, 255);

// UDP порт
const unsigned int localPort = 8888;
```

### Как изменить IP:

1. Откройте `arduino/quadcopter_stand.ino`
2. Найдите строку: `IPAddress ip(192, 168, 0, 177);`
3. Измените на нужный IP: `IPAddress ip(192, 168, 0, XXX);`
4. Загрузите код на Arduino
5. Обновите `config/config.json` (см. ниже)

## 🔧 Настройка Frontend

### В конфигурации (`config/config.json`):

```json
{
  "connection": {
    "mode": "direct",
    "direct": {
      "arduino_ip": "192.168.0.177",
      "arduino_port": 80,
      "endpoint": "/data"
    }
  },
  "arduino": {
    "network": {
      "mac": "DE:AD:BE:EF:FE:01",
      "ip": "192.168.0.177",
      "broadcast_ip": "192.168.0.255",
      "udp_port": 8888
    }
  }
}
```

### Как изменить IP:

1. Откройте `config/config.json`
2. Измените `"arduino_ip": "192.168.0.177"` на нужный
3. Измените `"ip": "192.168.0.177"` на нужный
4. Сохраните файл
5. Перезагрузите веб-страницу

## 🔧 Настройка Python Backend (опционально)

### В конфигурации (`config/config.json`):

```json
{
  "backend": {
    "udp": {
      "port": 8888
    },
    "http": {
      "host": "0.0.0.0",
      "port": 8080
    }
  }
}
```

Python сервер слушает UDP на порту 8888 и предоставляет HTTP API на порту 8080.

## 🌐 Требования к сети

### Для прямого подключения (Вариант 1):

```
Компьютер ←→ Роутер ←→ Arduino
   ↓                      ↓
192.168.0.X         192.168.0.177
```

**Требования:**
- Компьютер и Arduino в одной подсети (192.168.0.x)
- Маска подсети: 255.255.255.0
- Порт 80 открыт (HTTP)

### Для подключения через Python (Вариант 2):

```
Arduino → UDP Broadcast → Python Server → HTTP → Browser
  ↓                          ↓                      ↓
192.168.0.177          192.168.0.X            localhost:8000
```

**Требования:**
- Arduino и компьютер в одной подсети
- UDP порт 8888 открыт
- Python сервер запущен

## ✅ Проверка сети

### 1. Проверьте IP компьютера:

**Windows:**
```cmd
ipconfig
```

**macOS/Linux:**
```bash
ifconfig
# или
ip addr show
```

Должны видеть IP вида `192.168.0.X`

### 2. Проверьте доступность Arduino:

```bash
ping 192.168.0.177
```

Должен отвечать:
```
Reply from 192.168.0.177: bytes=32 time<1ms TTL=128
```

### 3. Проверьте HTTP сервер:

```bash
curl http://192.168.0.177/
```

Должен вернуть HTML страницу.

### 4. Проверьте /data endpoint:

```bash
curl http://192.168.0.177/data
```

Должен вернуть JSON с данными.

## 🐛 Проблемы с сетью

### Arduino не отвечает на ping

**Причины:**
1. Arduino не в той же подсети
2. Ethernet кабель не подключен
3. Ethernet модуль не работает
4. Firewall блокирует

**Решение:**
```bash
# Проверьте IP компьютера
ipconfig  # Windows
ifconfig  # macOS/Linux

# Проверьте что Arduino в той же подсети
# Компьютер: 192.168.0.X
# Arduino:   192.168.0.177

# Проверьте Serial Monitor Arduino
# Должно быть: {"status":"OK","sensor":"Ethernet"}
```

### Браузер не может подключиться

**Причины:**
1. Неверный IP в config.json
2. HTTP сервер не запущен
3. CORS проблемы
4. Firewall блокирует

**Решение:**
```bash
# 1. Проверьте доступность
curl http://192.168.0.177/data

# 2. Используйте тестовую страницу
http://localhost:8000/frontend/test.html

# 3. Проверьте консоль браузера (F12)
```

### UDP не работает

**Причины:**
1. Firewall блокирует UDP
2. Неверный broadcast адрес
3. Python сервер не слушает порт 8888

**Решение:**
```bash
# Проверьте что Python слушает UDP
netstat -an | grep 8888

# Проверьте Serial Monitor Arduino
# Должны видеть JSON сообщения каждые 200ms
```
