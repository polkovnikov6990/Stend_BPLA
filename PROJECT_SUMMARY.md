### 3. Реализовано два варианта подключения

#### Вариант 1: Прямое подключение (рекомендуется)

```
Browser → Arduino
   ↓         ↓
app.js → quadcopter_stand.ino
```

**Преимущества:**
- ✅ Простота
- ✅ Минимум зависимостей
- ✅ Прямая связь
- ✅ Низкая задержка

**Использование:**
```json
{
  "connection": {
    "mode": "direct"
  }
}
```

#### Вариант 2: Через Python сервер

```
Arduino → Python → Browser
   ↓        ↓        ↓
UART/UDP → web_server.py → app.js
```

**Преимущества:**
- ✅ Расширенный мониторинг
- ✅ Логирование
- ✅ Статистика
- ✅ GUI для управления

**Использование:**
```json
{
  "connection": {
    "mode": "server"
  }
}
```

### 4. Оптимизирован код Arduino

**`arduino/quadcopter_stand.ino`** - полнофункциональный скетч:

✅ **Поддержка всех датчиков:**
- Обороты через прерывания
- Температура с термисторов (уравнение Стейнхарта-Харта)
- Тензодатчики через HX711
- Акселерометр ADXL345 по I2C
- Микрофон через аналоговый вход

✅ **HTTP сервер:**
- Endpoint `/data` - JSON с данными
- Endpoint `/` - информационная страница
- CORS заголовки для браузера

✅ **Оптимизация:**
- Прерывания для подсчета оборотов
- Неблокирующее чтение датчиков
- Эффективное использование памяти

✅ **Конфигурация:**
- Все параметры из config.json
- Легкая адаптация под стенд
- Комментарии на русском
